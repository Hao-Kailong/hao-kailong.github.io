利用Scrapy爬取百度百科，互动百科

更新于 2020-02-29 | 分类于 Data Mining | 评论数：

项目简介

中文百科数据中蕴含了大量的知识，利用Scrapy爬取百科页面，为接下来的信息抽取等打下基础。

具体实践采用了Scrapy框架，这可能是python中最成熟的爬虫框架，利用代理ip池等技术，爬取全部的百科页面。

Scrapy架构

scrapy整体的结构如下图所示：

由Spider发送Request请求，Scheduler进行请求调度，Downloader执行具体的下载任务，并经过Pipeline写入数据库或磁盘等。用户可以自定义下载器中间件，比如在中间件中更换代理ip等，具有很大的简便性和灵活性。

爬取逻辑

百度百科

逻辑

由于百度百科的所有页面都是连续编码的，具有以下的格式：

http://baike.baidu.com/view/%s.html

例如， http://baike.baidu.com/view/1.html ，会被重定向为 https://baike.baidu.com/item/百度百科

所以，只需要连续的发送Request就可以爬取到全部的百科页面。

代理池

百度百科具有反爬虫策略，大量的请求会导致ip被封，因此需要采用代理ip技术

互动百科

逻辑

互动百科没有像百度百科那样的连续编码（在网上没有找到），因此只能换一种思路。

利用 http://fenlei.baike.com 作为起始页面，利用beautifulsoup提取所有的超链接，按照超链发现的方式进行图遍历，如果遇到满足 http://www.baike.com/wiki/ 的页面，则下载下来进入磁盘，否则继续遍历。

互动百科没有反爬虫策略，无需代理池
